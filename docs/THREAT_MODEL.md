# 部署威胁模型

## 保护目标

- 服务根外的主机文件不可通过 HTTP 访问；
- 未认证请求不可浏览、下载或修改文件；
- 只读用户不可取得写能力；
- 上传失败或中断不会产生可见半文件；
- 单个慢连接、昂贵搜索或大上传不会无限占用进程资源；
- 日志不泄露可复用凭据；
- 关闭和重启不会无限等待。

## 信任边界

```text
浏览器
  -> 私有 TLS 网关或反向代理
  -> Ram 的受限 HTTP/1.1 TCP 监听器
  -> Basic 认证、ACL 和资源准入
  -> 固定的服务根 dirfd
  -> 本地文件系统
```

网关负责 TLS、证书和域名。Ram 不解析转发来源头，因此直接代理会成为来源限流看到的
对端；用户级限制仍按认证账号独立执行。

## 攻击者能力

假定网络客户端可以发送畸形请求、超长头、慢请求体、断开的上传、路径编码边界和并发请求。
已认证写用户还可以选择文件名、目录层级、内容和操作顺序。上传内容可能是恶意的，Ram
只把它当作字节存储，不执行内容扫描。

## 可信假设

- 管理员、root、Ram 服务账号和网关管理员可信；
- 服务目录、配置、认证文件所在本地文件系统行为可靠；
- 服务运行期间挂载拓扑不会被特权进程改变；
- 其他本地进程不能直接写服务目录；
- 同一可写目录没有第二个 Ram 实例；
- 系统时间、内核随机源和 Linux 文件系统调用可用。

如果这些假设不成立，应通过容器、虚拟机、独立 Unix 账号和挂载权限建立更强隔离，而不是
关闭 Ram 的路径或资源检查。

## 主要缓解

路径请求经规范化后使用能力相对文件系统调用，拒绝穿越、越界链接、子挂载和特殊文件。
认证、ACL 和全局能力在开始读取或变更前检查。上传先进入不可见候选文件，只有全部检查
完成后才原子替换目标。

所有可能随输入增长的结构都有上限；网络和文件 I/O 有截止时间；日志使用有界队列。浏览器
预览限制实际读取字节、文本编码和主动内容能力，普通下载和 ZIP 使用原生流式响应。

## 明确不支持

- 直接面向不受信任网络的明文监听；
- 应用内 TLS 或可信转发来源头；
- 多实例共同写目录；
- WebDAV 客户端或系统挂载；
- 在线编辑和临时分享链接；
- 将服务目录用作不受信任多租户隔离边界。
